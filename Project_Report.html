Project Report: AI-Powered Investment Portfolio Analyzer

Cloud-Native Serverless System | Azure AI & Financial Intelligence Layer

1. Project Overview & Objective

The **Investment Portfolio Screenshot Analyzer** is a state-of-the-art solution designed to digitize, analyze, and store financial data from portfolio screenshots. The project leverages high-precision OCR technology and serverless cloud computing to automate what was previously a manual and error-prone process.

Primary Goal: To provide a seamless, 100% cloud-native experience where users can upload an image and instantly see a breakdown of their holdings, valuation metrics, and intelligent investment advice.

2. Key Features & Functionality

Feature 01 AI-Driven Portfolio Digitization

Uses Azure AI Vision (Read API) to extract data from unstructured screenshots. It identifies stock symbols (e.g., AAPL), share counts, and current market prices with over 95% accuracy.

Feature 02 Automated Financial Metrics (P/E & P/B)

The system automatically calculates/assigns Price-to-Earnings (P/E) and Price-to-Book (P/B) ratios for every holding. This allows users to quantify the value of their portfolio beyond just share price.

Feature 03 Intelligent Investment Suggestions

Proprietary logic evaluates the value metrics to provide a Decision Tag (Buy, Hold, or Avoid) for each stock. This transforms data into actionable financial advice.

Feature 04 Premium Cloud Dashboard

A fully responsive, high-end dashboard (Dark Mode, Glassmorphism) hosted on Azure Static Website. It provides drag-and-drop uploads and a historical view of all previous analyses.

3. Cloud Infrastructure Architecture

The solution is built on a distributed cloud architecture. The flow starts from the Azure Storage hosted frontend, triggers an Azure Function, which orchestrates the AI Vision OCR and persists analyzed data in Cosmos DB. All security is managed via Azure Key Vault and monitoring through Application Insights.

4. Full Technical Stack Breakdown

Service Name Project Implementation / Role
Azure AI Vision OCR engine used to identify and read text from complex portfolio images. Specifically used the 'Read' feature for mixed text extraction.
Azure Functions Serverless Python backend managing API requests, financial logic, and database integration. Functions as a REST API for the frontend.
Azure Cosmos DB NoSQL database used for permanent storage of analyzed portfolio records. Configured with a partition-key based on UserID for high-speed retrieval.
Azure Storage Hosts the front-end static website and provides Blob Storage for raw image logs and background trigger scenarios.
Azure Key Vault Secures sensitive credentials (DB keys, AI keys) from code exposure. Implements vault-based secret injection.
Application Insights Real-time monitoring of function performance, failure rates, and live telemetry to ensure 24/7 reliability.
Log Analytics Aggregates logs from across all services into a single searchable workspace for detailed technical auditing.
App Service Plan The underlying Linux-based Consumption plan that powers the serverless scaling and handles peak traffic automatically.

5. System Architecture & Workflow

  • Step 1: Upload — The user uploads a screenshot via the browser dashboard.
  • Step 2: API Trigger — The image is sent to an Azure Function HTTP endpoint.
  • Step 3: AI Inference — Function calls Azure Computer Vision to extract text blocks.
  • Step 4: Smart Parsing — Python regex logic isolates stock data from raw text.
  • Step 5: Valuation & Storage — Financial ratios are assigned, and data is saved to Cosmos DB.
  • Step 6: UI Update — Dashboard refreshes via AJAX to show results with suggestions.
